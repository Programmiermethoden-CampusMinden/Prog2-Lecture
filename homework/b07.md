# Zusammenfassung

Sie modellieren einen kleinen Zoo in Java.

Zunächst steht das Erstellen einer **Typhierarchie** mit *sealed classes* / *records* und der Einsatz von **Generics** im Vordergrund. Anschließend erweitern Sie dieses Modell um:

- sinnvolle **Zustandsänderungen** (Tiere aufnehmen, umsetzen, abgeben),
- das **Command-Pattern** mit Undo/Redo,
- **Logging** der ausgeführten Aktionen,
- sowie den Einsatz der **Stream-API**.

Ziel ist es, dass Sie die im Kurs behandelten Konzepte **eigenständig auf ein kleines, aber zusammenhängendes Beispiel anwenden**.

## Aufgabe 2: Command-Pattern für Zoo-Aktionen

In dieser Aufgabe führen Sie das **Command-Pattern** für typische Zoo-Aktionen ein.

### Aufgabe 2.1 Konkrete Commands auf dem Zoo

Definieren Sie ein generisches Interface:

```java
public interface Command<T> {
    void execute(T target);
    void undo(T target);
    String description();
}
```

`T` ist der Typ des "Empfängers" (Receiver), das wird in unserem Fall der `Zoo` sein.

Definieren Sie nun drei konkrete Commands, die auf einem `Zoo` arbeiten:

1.  `AddAnimalCommand`
    - Ziel: Ein neues Tier in ein bestimmtes Gehege aufnehmen
    - Attribute (Vorschlag):
        - ein konkretes Animal, z.B. `Cat`, `Primate`, ...
        - der Name des Zielgeheges (`String enclosureName`)
    - `execute(Zoo zoo)`:
        - Sucht das passende Gehege
        - Fügt das Tier der Bewohner-Liste hinzu (unter Berücksichtigung der Generics)
    - `undo(Zoo zoo)`:
        - Entfernt das hinzugefügte Tier wieder aus dem Gehege
2.  `MoveAnimalCommand`
    - Ziel: Ein Tier von einem Gehege in ein anderes umsetzen
    - Attribute (Vorschlag):
        - Name des Tiers (z.B. `String animalName`)
        - Name des Quellgeheges
        - Name des Zielgeheges
    - `execute(Zoo zoo)`:
        - Entfernt das Tier aus dem Quellgehege, und
        - fügt es dem Zielgehege hinzu
    - `undo(Zoo zoo)`:
        - Macht die Operation rückgängig (Tier zurück ins Ursprungsgehege)
3. `RemoveAnimalCommand`
    - Ziel: Ein Tier aus dem Zoo entfernen (z.B. Abgabe an einen anderen Zoo)
    - `execute(Zoo zoo)`:
        - Entfernt das Tier aus seinem Gehege
    - `undo(Zoo zoo)`:
        - Fügt es wieder in dasselbe Gehege ein

Speichern Sie in den Commands alle Informationen, die Sie für ein korrektes `undo` benötigen (z.B. Referenzen auf das ursprüngliche Gehege oder dessen Namen o.ä.).

### Aufgabe 2.2 Command-Manager mit Undo/Redo

Implementieren Sie nun die Klasse

```java
public class CommandManager<T> {
    private final Deque<Command<T>> undoStack = new ArrayDeque<>();
    private final Deque<Command<T>> redoStack = new ArrayDeque<>();

    public void executeCommand(Command<T> command, T target) { /* ... */ }
    public void undo(T target) { /* ... */ }
    public void redo(T target) { /* ... */ }
}
```

Arbeitsweise:

-   `executeCommand`:
    - Führt `command.execute(target)` aus
    - Legt das Kommando auf den `undoStack`
    - Leert den `redoStack`
- `undo`:
    - Wenn der `undoStack` nicht leer ist:
        - Nimmt das oberste Kommando
        - Ruft `undo(target)` darauf auf
        - Legt das Kommando auf den `redoStack`
- `redo`:
    - Analog zu `undo`, aber in umgekehrter Richtung (vom `redoStack` auf den `undoStack`)

Verwenden Sie den `CommandManager<Zoo>` in einem JUnit-Test, um eine Sequenz von Zoo-Aktionen auszuführen und mit Undo/Redo zu experimentieren.




## Aufgabe 3: Logging integrieren

Fügen Sie nun **Logging** mit `java.util.logging`hinzu, um alle Zoo-Aktionen nachvollziehbar zu machen.

Legen Sie dazu in `CommandManager` einen Logger an und loggen Sie in

- `executeCommand`:
    - `INFO`: Beginn der Ausführung (einschließlich `command.description()`)
    - `INFO` oder `FINE`: eine kurze Zusammenfassung des Zustands nach Ausführung (z.B. Anzahl Tiere im Zoo)
- `undo` und `redo`:
    - `INFO`: welches Kommando rückgängig gemacht bzw. erneut ausgeführt wurde
    - `WARNING`: wenn kein Undo/Redo möglich ist (Stack leer)

Führen Sie in Ihren konkreten Commands ebenfalls Logging hinzu:

- `WARNING`: wenn ein angefordertes Tier oder Gehege nicht gefunden wird
- `SEVERE`: bei schwerwiegenden Inkonsistenzen (falls Sie solche Fälle modellieren)

Demonstrieren Sie in einer Demo-Main, wie Sie das Log-Level für Ihre Logger gezielt umschalten können.
